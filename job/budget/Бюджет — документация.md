# Бюджетные заметки — устройство и дебаг

Карта системы: где что лежит, как считается, как отремонтировать любой блок если он сломался.

## TL;DR — где что лежит

| Артефакт           | Путь                                                             | Что делает                                                                                   |
| ------------------ | ---------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| **Шаблон**         | `Obsidian vault/Templates/Бюджет.md`                             | Эталонная структура заметки. Используется как референс + для ручной вставки через Templater. |
| **Генератор**      | `Obsidian vault/Scripts/budget_generator.py`                     | Создаёт `YYYYMM01 Бюджет.md`, заполняя frontmatter данными прошлой заметки.                  |
| **launchd-задача** | `~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist` | Запускает генератор 1-го числа в 09:00.                                                      |
| **Лог запусков**   | `~/Library/Logs/obsidian-budget.log`                             | stdout/stderr автозапусков (вне vault — не синкается).                                       |
| **Заметки**        | `Obsidian vault/Base/YYYYMMDD Бюджет.md`                         | Сами бюджетные заметки.                                                                      |
| **Эта дока**       | `Obsidian vault/Scripts/Бюджет — документация.md`                | Ты здесь.                                                                                    |

## Контракт frontmatter

Всё держится на YAML‑frontmatter заметки. Это **источник истины** — таблицы и графики ниже просто рендерят то, что там лежит. Меняешь числа только в YAML, не в таблицах.

```yaml
---
type: budget               # ← обязательный маркер, по нему Dataview ищет заметки
period: "2026-07"          # ← обязательный, YYYY-MM, по нему сортируется история
date: 2026-07-01
created: 2026-07-01 09:00
prev: "[[20260601 Бюджет]]"
usd_rub: 81.4              # ← курс для конверсии USD-счетов в ₽

assets:                    # ← список счетов: name, amount, currency, kind
  - {name: "Т Брокер", amount: 2060000, currency: "RUB", kind: "investment"}
  - {name: "Binance",  amount: 5400,    currency: "USD", kind: "crypto"}
  - {name: "Долг Мама", amount: -100000, currency: "RUB", kind: "debt"}

invested_deposits:         # ← накопительный учёт своих денег без рыночной маржи
  - {name: "Т Брокер", cumulative: 2005000, this_month: 0}

income:                    # ← плоский словарь категория → сумма ₽
  Зарплата: 0

expenses:                  # ← плоский словарь категория → сумма ₽
  Квартира: 0

plan:
  invest_target: 70000
  expenses_target: 200000
  savings_rate_target: 0.4
  nav_target: 0
---
```

**Допустимые `kind`** (важно — по ним считаются метрики):
- `investment` — брокерский, ИИС, всё что не крипта но «вложено»
- `crypto` — Binance, Bybit, Telegram, всё USD по умолчанию
- `liquidity` — накопительные счета, LQDT, депозиты
- `cash` — карты, кеш
- `debt` — долги (с отрицательным `amount` уменьшают СЧА)

Категории `investment + crypto` идут в «СЧА инвест-активов». `liquidity + cash` — в «Ликвидность». `debt` — отдельно.

## Команды на каждый день

```bash
SCRIPT="/Users/maximburdygin/Yandex.Disk.localized/Obsidian vault/Scripts/budget_generator.py"

# Сгенерировать заметку для текущего месяца (если запуск был пропущен)
python3 "$SCRIPT"

# Конкретный месяц
python3 "$SCRIPT" 2026-08

# Превью без записи в файл
python3 "$SCRIPT" 2026-08 --dry-run

# Перезаписать существующий файл (осторожно — затрёт твои правки!)
python3 "$SCRIPT" 2026-08 --force

# Посмотреть лог запусков (автоматических)
tail -50 ~/Library/Logs/obsidian-budget.log

# Проверить, что launchd-задача активна
launchctl print gui/$(id -u)/com.maximburdygin.obsidian-budget | head -20

# Перезагрузить задачу после правки .plist
launchctl bootout   gui/$(id -u)/com.maximburdygin.obsidian-budget
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist

# Запустить задачу прямо сейчас, не дожидаясь 1-го числа
launchctl kickstart -k gui/$(id -u)/com.maximburdygin.obsidian-budget

# Полностью отключить автоматику
launchctl bootout gui/$(id -u)/com.maximburdygin.obsidian-budget
```

## ❌ Дебаг — «не работает авто-создание заметки»

### Шаг 1. 1-го числа файл не появился — проверь, что задача загружена

```bash
launchctl print gui/$(id -u)/com.maximburdygin.obsidian-budget 2>&1 | head -20
```

Что искать:
- `state = waiting` или `state = not running` — задача загружена, ждёт триггера. **Норм.**
- `Could not find service` — задача **не загружена**. Загрузи:
  ```bash
  launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist
  ```

### Шаг 2. Задача загружена — но в 09:00 не сработала

Чаще всего:
- **Mac был выключен/спал в 09:00.** launchd запустит при ближайшем включении — обычно сразу при логине. Проверь время `created` в свежей заметке.
- **Файл уже существовал.** Скрипт без `--force` не перезаписывает. Лог покажет `[skip] … уже существует`.

```bash
tail -50 ~/Library/Logs/obsidian-budget.log
```

### Шаг 3. Лог пустой или с ошибкой

Запусти руками — увидишь ошибку сразу:

```bash
python3 "/Users/maximburdygin/Yandex.Disk.localized/Obsidian vault/Scripts/budget_generator.py" 2026-07 --dry-run
```

Типичные ошибки:

| Ошибка | Что чинить |
|---|---|
| `ModuleNotFoundError: No module named 'yaml'` | `/usr/bin/python3 -m pip install --user pyyaml` |
| `Не найдено /Users/.../Base` | Проверь путь к vault. Если переехал — поправь `VAULT` в начале скрипта. |
| `Ожидаю YYYY-MM` | Аргумент не в формате `2026-07`. |
| Лог содержит `Operation not permitted` | macOS блокирует доступ launchd к `~/Yandex.Disk.localized`. Дай Full Disk Access: System Settings → Privacy & Security → Full Disk Access → добавить `/usr/bin/python3`. |
| `state = not running` бесконечно | Удали `~/Library/LaunchAgents/...plist`, перезагрузи `launchctl bootout / bootstrap`. |

### Шаг 4. Скрипт работает руками, но не из launchd

Проверь `EnvironmentVariables` в plist — иногда launchd запускает в обрезанном окружении. Текущий plist уже задаёт `PATH` и `LANG`. Если меняешь — `plutil -lint` для валидации:

```bash
plutil -lint ~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist
```

### Шаг 5. Заметка создалась, но prev указывает в пустоту

Скрипт берёт **самую свежую** заметку с датой < 1-го числа целевого месяца, паттерн имени:
`^\d{8} Бюджет(?: \(N\))?\.md$`.

Если ты переименовал старую заметку и она перестала матчить паттерн — скрипт её не найдёт. Верни имя в формат или поправь регэксп `NOTE_RE` в начале скрипта.

## ❌ Дебаг — «не считаются нужные метрики» (Dataview не рендерит таблицы)

### Шаг 1. Проверь, что плагин включён

`Settings → Community plugins → Dataview` должен быть On. Если нет — включи и **перезапусти Obsidian**.

### Шаг 2. Проверь, что в frontmatter валидный YAML

Открой заметку и в правом меню выбери `View → Reading view` — Obsidian подсветит ошибки YAML.

Типичные косяки:
- Кавычки только частично экранированы: `name: "Долг "Мама""` → ломает парсер. Решение — экранируй или используй одинарные.
- Двоеточие в значении без кавычек: `Зарплата: 100: бонус` → ломается. Возьми в кавычки.
- Табы вместо пробелов в отступах. YAML требует пробелы.

Проверить YAML CLI’ом:

```bash
python3 -c "
import sys, yaml
text = open(sys.argv[1]).read()
fm = text.split('---', 2)[1] if text.startswith('---') else ''
print(yaml.safe_load(fm))
" "/Users/maximburdygin/Yandex.Disk.localized/Obsidian vault/Base/20260701 Бюджет.md"
```

### Шаг 3. Frontmatter валидный, но Dataview-блок пустой

Скорее всего блок упал на JS-ошибке. Открой DevTools: `Cmd+Option+I` → Console → перезагрузи заметку. Ошибка покажется красным.

Частые причины:
- `Cannot read properties of undefined (reading 'reduce')` — отсутствует ключ `assets` или `invested_deposits` в frontmatter. Скрипт всегда их кладёт, но если ты руками их удалил — добавь обратно хотя бы пустым списком: `assets: []`.
- `dv.pages('"Base"')` возвращает пусто — Dataview не индексирует папку. Проверь `Settings → Dataview → Inline JS queries` (включено), `Use legacy heading subpath` (не важно).

### Шаг 4. Метрика «Δ к прошлому месяцу» = `—`

Это значит, что в `Base/` нашлась только **одна** заметка с `type: budget`. История начинается со второй. До тех пор — `—` это норма.

Принудительно проверить, какие заметки видит Dataview:

```dataviewjs
const list = dv.pages('"Base"').where(p => p.type === "budget").sort(p => p.period, "asc");
dv.table(["Файл", "period", "СЧА total"],
  list.map(p => [
    p.file.link,
    p.period ?? "—",
    (p.assets || []).reduce((s,a) => s + (a.currency === "USD" ? a.amount*(p.usd_rub||81.4) : Number(a.amount||0)), 0).toLocaleString("ru-RU"),
  ])
);
```

Вставь временно в любую заметку — увидишь, что Dataview подцепил.

### Шаг 5. Старые legacy-заметки не учитываются в графиках

Это **специально**. Они без frontmatter, у них нет машинно-парсимого `type: budget`. Чтобы старые заметки попадали в графики, у каждой нужно добавить frontmatter руками. Минимальный набор:

```yaml
---
type: budget
period: "2026-05"
usd_rub: 81.4
assets:
  - {name: "Т Брокер", amount: 1902000, currency: "RUB", kind: "investment"}
  # ... все остальные
invested_deposits:
  - {name: "Т Брокер", cumulative: 1802000, this_month: 112000}
  # ...
---
```

Делать имеет смысл только за последние 3–6 месяцев, дальше история не критична.

### Шаг 6. СЧА считается, но не сходится с моими цифрами в legacy-заметках

Чаще всего разница из-за USD-конверсии:
- В старых заметках ты считал USD-доллары как «3.303.000 ₽ = 2.646.000₽ + 657.000₽» при курсе 81.4. Это `9000 × 81.4 ≈ 732 600`, не 657 000. Курс гулял.
- Скрипт берёт `usd_rub` из frontmatter каждой заметки и считает аккуратно. Если хочешь зафиксировать курс — поменяй `usd_rub` руками в свежей заметке.

## ❌ Дебаг — «не строятся графики»

### Шаг 1. Проверь, что плагин obsidian-charts включён

`Settings → Community plugins → Charts` должен быть On.

### Шаг 2. Графики падают на текстовую таблицу

Скрипт спроектирован с fallback: если `window.renderChart` недоступен, под графиком появляется таблица с теми же данными. Если ты видишь таблицу вместо графика — значит плагин не отдаёт API. Чини плагин (выключи/включи).

### Шаг 3. График пустой / `_Ещё нет заметок с frontmatter_`

Та же причина, что в Dataview-секции выше: нужно ≥ 1 заметка с `type: budget` для линейного графика и ≥ 2 для bar-графика. Запусти генератор для следующего месяца — обе появятся.

### Шаг 4. График есть, но линии скачут от какого-то месяца

Проверь, не стоит ли в конкретной заметке `usd_rub: 1` или другая опечатка. Найти быстро:

```bash
grep -H "^usd_rub:" "/Users/maximburdygin/Yandex.Disk.localized/Obsidian vault/Base/"*.md
```

## ❌ Дебаг — «legacy-парсер криво вытянул активы»

Скрипт парсит markdown-таблицы по regex. Если у legacy-заметки нестандартный формат (пробелы, опечатки, лишние колонки) — может ошибиться.

### Шаг 1. Сравни вход и выход

```bash
SCRIPT="/Users/maximburdygin/Yandex.Disk.localized/Obsidian vault/Scripts/budget_generator.py"
python3 "$SCRIPT" 2026-07 --dry-run | head -40
```

Сравни активы в выводе с таблицей в исходной заметке.

### Шаг 2. Типичные косяки legacy-парсера

| Проблема | Причина | Фикс |
|---|---|---|
| Счёт не подтянулся | Запись в строке `Summary` / `Total` — пропускается фильтром | Если у тебя счёт случайно назван `Summary` — переименуй |
| Сумма = 0 у счёта со значением | Число с экзотическим разделителем (`2 060 000`) — пропускается | Поправь руками в свежей заметке (всё равно перетрёшь). |
| Перепутана валюта | Колонка currency пустая или с `**$**` | Поправь руками |
| Неверный `kind` | Эвристика по названию: «брок/иис» → investment, «карт» → cash, «долг» → debt | Перепиши `kind` руками в YAML |

### Шаг 3. После того как один раз правильно заполнил frontmatter, парсер больше не нужен

Скрипт сначала пытается читать `type: budget` frontmatter. Если он есть — legacy-парсер не зовётся. Так что один раз выровнял свежий месяц — дальше всё через frontmatter, чисто и предсказуемо.

## Типовые задачи

### Добавить новый счёт

В свежей заметке в `assets` дописать строку:

```yaml
  - {name: "Точка Брокер", amount: 50000, currency: "RUB", kind: "investment"}
```

Скрипт при следующей генерации перенесёт его в новый месяц автоматически.

### Добавить новую категорию расходов

В `expenses` дописать ключ. Можно прямо нулевой. Скрипт при следующей генерации перенесёт ключ (но обнулит значение, как и все остальные).

```yaml
expenses:
  Квартира: 0
  Образование: 0   # ← новая
```

### Изменить время автозапуска

Открой `~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist`, поменяй `Hour`/`Minute`/`Day`, перезагрузи:

```bash
launchctl bootout   gui/$(id -u)/com.maximburdygin.obsidian-budget
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.maximburdygin.obsidian-budget.plist
```

### Сменить vault (если папка переехала)

Поправь две строки:
1. В `budget_generator.py` константу `VAULT` в самом начале.
2. В plist пути `ProgramArguments`, `WorkingDirectory`.

Затем перезагрузи launchd.

### Сделать ручную правку прошлого месяца, чтобы графики стали точнее

Любой backfill старых заметок — просто добавить им frontmatter с обязательным минимумом: `type: budget`, `period: "YYYY-MM"`, `assets`, `invested_deposits`. Дальше Dataview подхватит автоматически.
